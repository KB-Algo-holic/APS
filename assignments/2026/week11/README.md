# 그래프(Graph)란

그래프는 **정점(Vertex)** 과 **간선(Edge)** 으로 이루어진 자료구조입니다.

예시

```
A --- B
|     |
C --- D
```

구성 요소

| 구성 | 의미 |
|------|------|
| 정점(Vertex) | 노드 |
| 간선(Edge) | 연결 관계 |

그래프는 "연결 관계"를 표현하는 자료구조입니다.

예시

- 사람 간 친구 관계
- 도시 간 이동 경로
- 컴퓨터 네트워크 연결
- SNS 팔로우 관계

---

# 그래프의 종류

## 방향 그래프 (Directed Graph)

간선에 방향 존재

```
A → B
```

A에서 B로 이동 가능

B에서 A로 이동 불가능

예시

- 팔로우 관계
- 작업 순서
- 위상정렬 문제

---

## 무방향 그래프 (Undirected Graph)

양방향 이동 가능

```
A — B
```

예시

- 친구 관계
- 도로 연결
- 네트워크 연결

---

## 가중치 그래프 (Weighted Graph)

간선에 비용 존재

```
A --5--> B
```

예시

- 거리
- 시간
- 비용

최단거리 문제에서 사용

# 그래프 저장 방식

그래프는 대표적으로 두 가지 방식으로 저장합니다

| 저장 방식 | 특징 |
|----------|------|
| 인접행렬 | 구현 간단 |
| 인접리스트 | 메모리 효율 |

---

# 인접행렬 (Adjacency Matrix)

## 개념

2차원 배열을 사용하여 연결 여부 저장

예시

```
  A B C
A 0 1 1
B 1 0 0
C 1 0 0
```

의미

A와 B 연결됨

A와 C 연결됨

---

# 인접행렬 특징

장점

- 구현 쉬움
- 연결 여부 확인 빠름
- 직관적 구조

단점

- 메모리 많이 사용

공간복잡도

```
O(V²)
```

(V = 정점 개수)

---

# 인접행렬 구현

선언

```java
int[][] graph = new int[n][n];
```

간선 추가

```java
graph[a][b] = 1;
graph[b][a] = 1;
```

(무방향 그래프)

---

# 연결 여부 확인

```java
if (graph[a][b] == 1)
```

시간복잡도

```
O(1)
```

---

# 가중치 그래프 표현

```java
graph[a][b] = weight;
```

---

# 인접리스트 (Adjacency List)

## 개념

각 정점마다 연결된 정점 목록 저장

예시

```
A → B, C
B → A
C → A
```

구조

```
0 → 1, 2
1 → 0
2 → 0
```

---

# 인접리스트 특징

장점

- 메모리 효율적
- 실제 코딩테스트에서 가장 많이 사용

공간복잡도

```
O(V + E)
```

(V = 정점 개수, E = 간선 개수)

단점

- 연결 여부 확인 느림

시간복잡도

```
O(V)
```

---

# 인접리스트 구현

선언

```java
ArrayList<ArrayList<Integer>> graph = new ArrayList<>();
```

초기화

```java
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}
```

간선 추가

```java
graph.get(a).add(b);
graph.get(b).add(a);
```

(무방향 그래프)

---

# 가중치 그래프 인접리스트

```java
ArrayList<ArrayList<int[]>> graph = new ArrayList<>();
```

예시

```
[node, weight]
```

삽입

```java
graph.get(a).add(new int[]{b, weight});
```

---

# 인접행렬 vs 인접리스트 비교

| 항목 | 인접행렬 | 인접리스트 |
|------|----------|------------|
| 공간복잡도 | O(V²) | O(V+E) |
| 연결 확인 | 빠름 | 느림 |
| 구현 난이도 | 쉬움 | 보통 |
| 사용 빈도 | 낮음 | 매우 높음 |

코딩테스트에서는 대부분

```
인접리스트 사용
```

---

# 언제 인접행렬을 사용할까

이 경우 사용

- 정점 개수가 작음
- 연결 여부 확인이 많음
- 완전 그래프

예시

```
n ≤ 500
```

---

# 언제 인접리스트를 사용할까

이 경우 사용

- 정점 개수가 많음
- 간선 개수가 적음
- DFS / BFS 문제

예시

```
n ≤ 100,000
```

# 그래프 입력 처리 패턴

입력 예시

```
n m
a b
a b
a b
```

구현

```java
for (int i = 0; i < m; i++) {
    int a = sc.nextInt();
    int b = sc.nextInt();

    graph.get(a).add(b);
    graph.get(b).add(a);
}
```

## 방향 그래프인지 확인 필수

잘못 구현하면 오답 발생

```

```
