# 3월 1주차 - 기본 구현 (Implementation)

## 목표
알고리즘 문제 풀이의 가장 기본이 되는 **구현 능력**을 기르는 주차입니다.  
문제에서 요구하는 조건을 정확히 해석하고 코드로 그대로 옮기는 연습을 합니다.

구현 문제는 특별한 알고리즘보다 **문제 이해력과 꼼꼼함**이 더 중요합니다.

---

## 주요 문제 유형

### 1️⃣ 조건 기반 상태 변화
주어진 명령 또는 조건에 따라 값을 변경하는 유형

예시
- 캐릭터 이동
- 점수 계산
- 명령어 기반 상태 변화

핵심 포인트
- 조건 분기 (if / switch)
- 반복문 처리

---

### 2️⃣ 좌표 이동 / 방향 이동
2차원 좌표에서 이동을 구현하는 문제

예시
- 상하좌우 이동
- 방향 전환
- 격자 이동

핵심 패턴
dx = [-1, 1, 0, 0]
dy = [0, 0, -1, 1]
---

### 3️⃣ 입력 기반 시뮬레이션
주어진 입력 순서대로 상태를 업데이트

예시
- 게임 진행
- 명령어 처리
- 점수판 계산

---

## 풀이 원리

구현 문제는 일반적으로 아래 단계로 풀이합니다.

### 1️⃣ 문제를 상태로 정의

예시
현재 위치
현재 점수
현재 방향

### 2️⃣ 명령을 하나씩 처리
for command in commands:
상태 변경
### 3️⃣ 결과 반환

---

## 구현 문제 접근 방법

### Step 1
문제를 읽고 **필요한 상태 변수 정의**

### Step 2
입력 또는 명령을 **순서대로 처리**

### Step 3
조건에 맞게 **상태 업데이트**

---

## 꿀팁

### 💡 1. 문제를 코드로 바로 짜지 말 것
먼저 종이에 **상태 변화 흐름**을 정리하면 구현이 쉬워집니다.

---

### 💡 2. 변수 이름을 명확하게

❌ 나쁜 예
int a

좋은 예
int currentPosition 
int score

---

### 💡 3. 작은 단위 테스트

중간중간 출력해보면서 상태 확인
System.out.println(x + " " + y)

---

### 💡 4. 문제 조건 체크

구현 문제는 다음 실수가 많습니다

- 인덱스 오류
- 조건 누락
- 초기값 오류
