# 3월 3주차 - 시뮬레이션 (Simulation)

## 목표
시뮬레이션 문제는 **문제에서 주어진 상황을 그대로 코드로 구현하는 유형**입니다.  
주로 **격자 이동, 게임 진행, 상태 변화** 등의 문제가 등장합니다.

이 유형은 코딩 테스트에서 **구현 + 자료구조 + 문제 해석 능력**을 동시에 요구하는 대표적인 문제입니다.

---

# 주요 문제 유형

## 1️⃣ 격자(Grid) 이동 시뮬레이션

2차원 배열에서 캐릭터 또는 객체가 이동하는 문제입니다.

예시
- 로봇 이동
- 상하좌우 이동
- 방향 전환

핵심 패턴
int[] dx = {-1, 1, 0, 0};
int[] dy = {0, 0, -1, 1};

좌표 이동
x += dx[dir];
y += dy[dir];

---

## 2️⃣ 상태 변화 시뮬레이션

시간 또는 명령에 따라 상태가 변하는 문제입니다.

예시
- 게임 진행
- 턴 기반 문제
- 명령어 실행

예시 구조
for (command : commands) {
상태 업데이트
}

---

## 3️⃣ 시간 기반 시뮬레이션

시간이 흐르면서 상황이 변하는 문제입니다.

예시
- 초 단위 진행
- 이벤트 발생
- 순차 처리

예시 구조
while (time < end) {
상태 변화
time++
}

---

# 시뮬레이션 풀이 원리

시뮬레이션 문제는 다음 단계로 접근합니다.

## Step 1 상태 정의

문제를 읽고 필요한 상태를 정의합니다.

예시
현재 위치
현재 방향
현재 시간
현재 점수

---

## Step 2 규칙 정의

문제에서 주어진 규칙을 코드로 변환합니다.

예시
if (벽이면 이동하지 않음)
if (아이템이면 점수 증가)

---

## Step 3 반복 수행

주어진 조건 동안 시뮬레이션 진행
while (조건) {
상태 변경
}
---

# 시뮬레이션 문제 접근 방법

### 1️⃣ 문제를 단계별로 분해

예시

1. 이동

2. 충돌 체크
 
3. 상태 변경

---

### 2️⃣ 조건을 코드로 그대로 옮기기

문장을 코드로 번역한다고 생각하면 쉽습니다.

예

문제 설명

> 벽이면 이동하지 않는다

코드
if (board[nx][ny] == WALL)

---

### 3️⃣ 순서를 절대 바꾸지 않기

시뮬레이션 문제는 **순서가 매우 중요합니다**

예
이동 → 충돌 체크 → 상태 변경

---

# 꿀팁

## 💡 1. 문제를 먼저 그림으로 그리기

격자 문제는 종이에 그리면 훨씬 이해가 쉽습니다.

---

## 💡 2. 상태를 구조화하기

복잡한 문제는 클래스로 관리하면 좋습니다.
class Robot {
int x;
int y;
int dir;
}

---

## 💡 3. 좌표 범위 체크 먼저

시뮬레이션 문제에서 가장 많이 발생하는 오류

---

## 💡 3. 좌표 범위 체크 먼저

시뮬레이션 문제에서 가장 많이 발생하는 오류
nx >= 0 && ny >= 0
nx < n && ny < m

---

## 💡 4. 작은 테스트로 검증

문제가 복잡하면 작은 입력으로 테스트합니다.

---

# 자주 쓰는 패턴

## 방향 배열
int[] dx = {-1, 1, 0, 0};
int[] dy = {0, 0, -1, 1};

---

## 방향 회전
dir = (dir + 1) % 4;

---

## 좌표 이동
nx = x + dx[dir];
ny = y + dy[dir];
